# Miller Gold Anatomy v1 Corpus (Local Copy for CasePrep RAG Phase 1)

**Source**: `/Users/alexbaur/anatomy_pipeline/data/anatomy_miller_chapter/anatomy_gold_v1_pinecone_ready.jsonl` (authoritative gold corpus from anatomy_pipeline project)

**Record count**: 717

**Corpus version**: gold_v1 (from `corpus_version` field in records; `source: "miller"`)

**Schema** (pinecone-ready slim export):
- `id`: stable identifier (e.g. `miller-canonical-p1-8f0fba39579b`)
- `text`: embedding/display text (canonical_fact)
- `source_quote`: exact excerpt from Miller source
- `page`: source_page (int)
- `section_path`, `heading`: provenance
- `source`, `corpus_version`
- `metadata`: {
    "specialty", "region", "subregion", "diagnosis1", "procedure",
    "anatomy_terms" (dict of lists), "structures_at_risk" (list),
    "approach_terms" (list), "case_associations" (list),
    "quality_tier" ("high" | "medium" | "low_but_acceptable"),
    "metadata_trust"
  }

**Intended use (Phase 1)**:
- Local-only anatomy retrieval for BroBot /case-prep RAG.
- Embed with `text-embedding-3-small` (to match production CasePrep embedding model).
- Retrieve Miller-backed anatomy facts, structures-at-risk, landmarks, etc. for case queries.
- Feed into dedicated anatomy context builder (no hallucination; only from retrieved chunks).
- Behind `ENABLE_LOCAL_ANATOMY_RAG=true` flag.
- When flag off or absent: fall back to existing `anatomy_gpt.py` (approach catalog + GPT).

**Important**:
- This is a **read-only local copy** for development/testing in CasePrep.
- **Do not modify** the JSONL or original in anatomy_pipeline.
- **No Pinecone upload** in Phase 1. (See reports/local_anatomy_rag_phase1_implementation.md for Phase 2 plans using namespace.)
- Gold corpus was QC'd (line audits, human packet review, holdout) for source fidelity and ortho utility. Prefer high/medium quality_tier.
- 717 records is high-precision but small coverage; expect `limitedAnatomyContext` for some edge cases.
